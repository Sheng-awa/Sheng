# server/ — 后端预留目录

这里是给以后的后端预留的位置，目前网站是纯静态的，这个目录只起占位作用。

## 设想中的用法

- 放 API 服务代码（比如 Node.js + Express / Hono，或者你喜欢的任何框架）
- 前端通过 `/api/*` 路径访问后端接口
- 以后想加的功能（博客、留言板、日记等）都可以从这里长出来

## 与 Vercel 的关系

- 如果走 **Vercel Serverless Functions**：可以直接在仓库根目录建 `api/` 目录
  写函数（这是 Vercel 的约定），本目录则可以放共享逻辑
- 如果走**独立后端服务**：本目录就是后端项目根目录，到时候在这里
  初始化 package.json 即可

现在不用动它，安心当前端小窝就好 ✿
